{ ES6 } - Classes

            
              class Car {
                constructor(name) {
                  this.brand = name;
                }
              
                present() {
                  return 'I have a ' + this.brand;
                }
              }
              
              class Model extends Car {
                constructor(name, mod) {
                  super(name);
                  this.model = mod;
                }  
                show() {
                    return this.present() + ', it is a ' + this.model
                }
              }
              mycar = new Model("Ford", "Mustang");
              mycar.show();

{ ES6 } Variables

            
              
              hello = () => {
                if(1===1) {
                  var a = 10;
                }
                console.log(a); // Would print the value 10
              }

              
              hello = () => {
                if(1===1) {
                  let a = 10;
                }
                console.log(a);  // ReferenceError - Can't find variable 'a'
              }

              
              hello = () => {
                const a = 10;
                a = 20; // TypeError: Attempted to assign to readonly property.
              }

Basics - States & Lifecycle

  • Two types of data in React / React Native Components.
  • `props` and `state`. Both are Javascript Objects
  • State requires a default value to start with.
  • State changes are the events that calls the re-rendering of the component
  • Keep the state information as minimal as possible. don't keep the computation or other react components in the State Object
  • State typically has 3 actions
    • setting default state getInitialState
    • getting current state this.state
    • updating current state this.setState

Props Vs States

  • we can access them via this.
  • both can have initial value
  • both are just javascript objects.
  • props are immutable. they cannot be changed. but state can be changed.
  • state can change, yet, it can only be mutated from the class where it originated from.

Lifecycle of a Component

  • Lifecycle Includes
    • Initialization
      • Constructor
    • Mounting
      • componentWillMount()
      • render()
      • componentDidMount()
    • Updating
      • componentWillReceiveProps()
      • shouldComponentUpdate()
      • componentWillUpdate()
      • render()
      • setState()
      • componentDidUpdate()
    • Unmounting
      • componentWillUnmount()
